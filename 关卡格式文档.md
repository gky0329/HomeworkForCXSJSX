# 关卡格式文档 — 贡献者指南

> 无需懂 Python，只需写 JSON。复制模板 → 填空 → 发布。

---

## 快速开始（3 步发布新关卡）

### 第 1 步：复制模板

```bash
cp data/levels/_template.json data/levels/oop_NNN.json
```

### 第 2 步：修改内容

用任意文本编辑器打开 `oop_NNN.json`，修改以下区域：

| 字段 | 说明 | 示例 |
|------|------|------|
| `id` | 唯一标识，格式 `oop_数字` | `"oop_003"` |
| `title` | 关卡标题 | `"访问控制：public 与 private"` |
| `unit` | 所属学习单元 | `1` |
| `order` | 单元内顺序 | `3` |
| `prerequisites` | 前置关卡 id 列表 | `["oop_001", "oop_002"]` |
| `initial_code` | 预填代码，`____` 为挖空处 | 见下方示例 |
| `blanks` | 每个挖空处的定义 | 见下方示例 |
| `goal.checks` | 通关判定条件 | 见 §4 |
| `hints` | 3 条渐进提示 | 见 §6 |

### 第 3 步：注册到索引

在 `data/levels/index.json` 的 `levels` 数组中添加一行：

```json
{ "id": "oop_003", "title": "访问控制：public 与 private", "type": "challenge", "unit": 1, "order": 3, "prerequisites": ["oop_001", "oop_002"] }
```

完成。不会改动任何 Python 代码。

## 一、完整 JSON 结构

```json
{
  "id": "oop_NNN",
  "title": "关卡标题",
  "type": "challenge",
  "unit": 1,
  "order": 1,
  "prerequisites": [],

  "npc_dialogue": [
    {"npc": "villager", "text": "NPC 说的话..."},
    {"npc": "iron_golem", "text": "NPC 说的话..."}
  ],

  "theory": "教学文本 150-400 字。结构：类比引入 → 概念定义 → OOP 行为 → 本关任务。",
  "key_points": ["知识点 1", "知识点 2"],

  "initial_code": [
    "____ Player {",
    "public:",
    "    int health;",
    "};",
    "Player p;",
    "p.____ = 20;"
  ],

  "blanks": [
    {
      "line": 0,
      "type": "text_input",
      "placeholder": "class",
      "width_chars": 10,
      "validator": "cpp_keyword"
    }
  ],

  "goal": {
    "checks": [
      {"type": "class_exists", "name": "Player"},
      {"type": "instance_exists", "var_name": "p", "class_name": "Player"},
      {"type": "member_value_equals", "var_name": "p", "member": "health", "value": 20},
      {"type": "no_errors"}
    ]
  },

  "hints": [
    "方向性提示（不透露答案）",
    "工具性提示（提示语法元素）",
    "模板性提示（给出语句模板）"
  ],

  "success_message": "通关文案。解释 Canvas 上发生了什么。",
  "failure_guidance": "失败时的补充说明。",
  "estimated_time_min": 5
}
```

## 二、字段说明

### 必填字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 唯一标识，建议 `oop_NNN` |
| `title` | string | 关卡标题，显示在页面顶部 |
| `type` | string | 固定为 `"challenge"` |
| `unit` | int | 学习单元编号，从 1 开始 |
| `order` | int | 单元内顺序，从 1 开始 |
| `prerequisites` | list | 前置关卡 id，无则 `[]` |
| `initial_code` | list[string] | 预填代码行，`____` 表示挖空位置 |
| `blanks` | list[object] | 挖空定义，见 §3 |
| `goal.checks` | list[object] | 通关条件，见 §4 |

### 推荐字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `theory` | string | 教学文本，150-400 字 |
| `key_points` | list[string] | 2-5 条关键知识点 |
| `hints` | list[string] | 3 条渐进提示 |
| `success_message` | string | 通关时显示的文案 |
| `estimated_time_min` | int | 预计完成时间（分钟） |

### 可选字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `npc_dialogue` | list | NPC 对话，无则跳过课前阅读环节 |
| `failure_guidance` | string | 失败时的补充说明 |
| `code_line_notes` | list[string] | 逐行代码注释，与 `initial_code` 行数一致 |

### NPC 对话格式

```json
"npc_dialogue": [
  {"npc": "villager",    "text": "村民的台词"},
  {"npc": "iron_golem",  "text": "铁傀儡的台词"}
]
```

支持的 NPC 类型：`villager`（村民，棕色）、`iron_golem`（铁傀儡，银色）。

## 三、挖空定义 (blanks)

每个 `____` 在 `initial_code` 中占 4 个字符。`blanks` 按代码中 `____` 出现的先后顺序定义。

```json
{
  "line": 0,                    // 所在行号（从 0 开始）
  "type": "text_input",         // 固定为 "text_input"
  "placeholder": "class",       // 输入框占位提示
  "width_chars": 10,            // 输入框宽度（字符数）
  "validator": "cpp_keyword"    // 输入校验规则，见下表
}
```

### Validator 可选值

| Validator | 允许输入 | 用途示例 |
|-----------|---------|---------|
| `cpp_keyword` | `class`, `struct` | 类定义关键字 |
| `cpp_access` | `public`, `private`, `protected` | 访问控制 |
| `identifier` | 字母/数字/下划线 | 类名、成员名、变量名 |
| `number` | 数字 0-9 | 成员值 |
| `any` | 任意字符 | 自由输入（谨慎使用） |

---

## 四、通关判定 (goal.checks)

`checks` 是一个列表，**所有条件同时满足**才算通关。

### 可用判定类型

| type | 参数 | 说明 | 示例 |
|------|------|------|------|
| `class_exists` | `name` | 某类是否被定义 | `{"type": "class_exists", "name": "Player"}` |
| `class_has_member` | `class_name`, `member_name` | 某类是否有某成员 | `{"type": "class_has_member", "class_name": "Player", "member_name": "health"}` |
| `instance_exists` | `var_name`, `class_name` | 某变量是否是某类的实例 | `{"type": "instance_exists", "var_name": "p", "class_name": "Player"}` |
| `member_value_equals` | `var_name`, `member`, `value` | 某实例的某成员是否为指定值 | `{"type": "member_value_equals", "var_name": "p", "member": "health", "value": 20}` |
| `instance_count` | `class_name`, `count` | 某类的实例数量 | `{"type": "instance_count", "class_name": "Creeper", "count": 2}` |
| `no_errors` | 无 | 无任何错误（如访问越权、重复定义） | `{"type": "no_errors"}` |

### 典型 check 组合

**基础：定义类 + 创建实例 + 赋值**

```json
"checks": [
  {"type": "class_exists", "name": "Player"},
  {"type": "instance_exists", "var_name": "p", "class_name": "Player"},
  {"type": "member_value_equals", "var_name": "p", "member": "health", "value": 20},
  {"type": "no_errors"}
]
```

**多实例：两个类 + 数量校验**

```json
"checks": [
  {"type": "class_exists", "name": "Creeper"},
  {"type": "class_exists", "name": "IronGolem"},
  {"type": "instance_count", "class_name": "Creeper", "count": 2},
  {"type": "instance_count", "class_name": "IronGolem", "count": 1},
  {"type": "no_errors"}
]
```

**访问控制：private 成员不允许外部赋值**

```json
"checks": [
  {"type": "class_exists", "name": "Player"},
  {"type": "instance_exists", "var_name": "p", "class_name": "Player"},
  {"type": "no_errors"}
]
```

## 五、代码行注释 (code_line_notes)

可选字段。长度必须与 `initial_code` 一致，空行也占一个位置。

```json
"initial_code": [
  "class Player {",
  "public:",
  "    int health;",
  "};",
  "Player p;"
],
"code_line_notes": [
  "定义一个名为 Player 的类 ← 本关新概念",
  "以下成员可被外部访问",
  "声明一个 int 类型的 health 成员",
  "",
  "创建 Player 类的实例 p"
]
```

---

## 六、渐进式提示 (hints)

3 条提示，用户每点一次「需要提示？」展开一条。

| 级别 | 类型 | 要求 | 示例 |
|------|------|------|------|
| 提示 1 | 方向性 | 引导思考，不透露答案 | `"你需要一个关键字来告诉编译器你正在定义一个自定义类型"` |
| 提示 2 | 工具性 | 提示用到的语法 | `"用 class 关键字，类体用 {} 括起来"` |
| 提示 3 | 模板性 | 给出完整语句 | `"class Player {\npublic:\n    int health;\n};"` |

---

## 七、设计原则速查

- **单一概念**：每关只引入 1 个新语法元素。如果同时教 `class` 和 `public/private`，分两关。
- **视觉锚点**：`{}`、`;`、缩进等非教学语法全部预填，只挖空本关的新关键字。
- **150-400 字 theory**：类比引入(50字) → 概念定义(100字) → OOP 行为(100字) → 本关任务(50字)。
- **类比优先**：用「蓝图 vs 房屋」比「类 vs 对象」更直观。

---

## 八、自检清单

发布前逐项确认：

- [ ] `id` 唯一，不与已有关卡重复
- [ ] `initial_code` 中 `____` 的数量 = `blanks` 数组长度
- [ ] 每个 blank 的 `line` 与实际行号一致
- [ ] `prerequisites` 正确（无则 `[]`）
- [ ] `goal.checks` 最后一个 check 是 `{"type": "no_errors"}`
- [ ] `code_line_notes` 行数 = `initial_code` 行数（可选字段，但建议写）
- [ ] `hints` 有 3 条，级别递进
- [ ] 已在 `index.json` 中注册
