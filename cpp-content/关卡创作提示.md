# 关卡创作提示 — C++ OOP 教学工具

> 你是一位精通 C++ OOP 的内容设计师。请根据本文档的要求，产出 `data/levels/` 目录下的关卡 JSON 文件。
>
> **目标**：产出高质量、情景化的 C++ 类与对象教学关卡，内容以 JSON 格式交付。

---

## 一、世界观

### 核心故事

村民**小C**（一个年轻的 Minecraft 村民）在矿洞深处捡到了一块发着微光的**黑色方块**。这个方块不同于他见过的任何东西——它似乎有自己的意识，会随着小C的情绪改变颜色。

小C颤抖着手把它带回家，在墙角给它铺了一块软布，小心翼翼地命名它为**「家家」**。

他没有想到，这个决定彻底改变了他的人生。

### 叙事主线（按关卡推进）

| 关卡 | 剧情场景 | 对应的 OOP 概念 |
|------|---------|----------------|
| 1 | 小C捡到黑色方块，命名「家家」 | **class 定义 + 对象创建**：家家的身份需要一张「图纸」来描述 |
| 2 | 家家展现属性：会发光、会变热、会变冷 | **成员变量**：家家有颜色、温度、能量值 |
| 3 | 小C尝试触摸家家的核心，手被弹开 | **public vs private**：有些属性可以碰，有些不行 |
| 4 | 家家开始回应小C：用闪烁回应呼唤 | **成员函数**：方块不只是数据，它还能做事情 |
| 5 | 小C发现了家家的「制造图纸」 | **构造函数**：家家是如何被初始化出来的 |
| 6 | 出现了红方块(炽焰)和蓝方块(寒冰) | **继承**：不同的方块，共享同一张基础图纸 |

### 角色设定

| 角色 | 身份 | 语气 | 在故事中的作用 |
|------|------|------|-------------|
| **小C** | 村民，故事主角 | 好奇、谨慎、逐渐自信 | 用户代入的角色，用户的代码 = 小C的操作 |
| **铁傀儡** | 小C的朋友和保镖 | 沉稳、保护性、偶尔幽默 | 技术注释者，用行为暗示概念 |
| **村民A** | 旁观的村民 | 大惊小怪、问傻问题 | 引出概念解释（他在问，铁傀儡在答） |
| **家家** | 黑色方块实体 | 无声，通过视觉变化表达 | 叙事中心，Canvas 上的主角 |

### 情景化的铁律

- ❌ NPC 不讲课。不讲「类就像蓝图」这种干瘪话
- ✅ 角色在情境中说话。铁傀儡：「小C，你得给家家写一张图纸。不然别人怎么知道你手里这个方块和地上的石头不一样？」
- ✅ 知识点通过**角色互动**传达，而非 theory 文本告诉用户
- ✅ `theory` 字段保留，但只写 50-80 字的**本关故事摘要**，不写概念讲授

## 二、题目设计要求

### 2.1 填空题的叙事理由

每个 `____` 必须在剧情中有**操作理由**，不能因为是知识点就挖掉：

| 要挖的词 | ❌ 坏理由 | ✅ 好理由 |
|----------|----------|----------|
| `class` | 因为要教 class 关键字 | 「小C需要给家家建一个『类别档案』，告诉世界家家是什么种类的东西」 |
| `public` | 因为是访问控制 | 「铁傀儡提醒小C：只有标记为公共的属性，村民才能看到」 |
| `health` | 因为这是成员变量 | 「家家今天有点虚弱，小C想查看它的能量值」 |

### 2.2 每关的填空分布

- 每关 **2-4 个空白**，不超过 4 个
- 至少 1 个空白需要理解代码上下文才能填对，不能靠 hint 照抄
- 最后一个空白的答案应该在之前几关中以其他形式出现过（**复用已有知识**）

### 2.3 错误反馈要有叙事感

| 错误类型 | ❌ 旧反馈 | ✅ 新反馈 |
|----------|----------|----------|
| 访问 private 成员 | 成员 health 是 private 的，外部不能访问 | 「小C的手被家家的外壳弹开了！铁傀儡说：『它把核心藏起来了，只有家家的内部操作才能碰那里。』」 |
| 类名拼错 | 类 Plaeyr 不存在 | 「图纸上的名字写错了——小C看了看，发现他写的是 Plaeyr 而不是 Player」 |
| 赋值类型错误 | 未提供 | 「小C想给家家的温度赋值 '很烫' ——但温度得是数字才行啊！」 |

### 2.4 难度递进

| 关卡 | 新概念 | 提示强度 | 用户行为复杂度 |
|------|--------|---------|--------------|
| 1 | class 定义 + 创建对象 | 3 条提示全保留 | 低（填 2 词） |
| 2 | 成员变量 + 赋值 | 提示 3 收紧为半模板 | 中（填 3 词，1 词需自主推断） |
| 3 | public/private 访问控制 | 只出提示 1 和 2，无模板 | 中高（填 2 词，1 词预测后果） |
| 4 | 成员函数 | 只出提示 1，方向性 | 高（填 3 词，需独立完成） |

## 三、题目格式规范

### 3.1 完整关卡 JSON 骨架

```json
{
  "id": "oop_001",
  "title": "【必填】关卡标题 — 建议用叙事句，如「捡到一块发光的石头」而非「class 关键字」",
  "type": "challenge",
  "unit": 1,
  "order": 1,
  "prerequisites": [],

  "npc_dialogue": [
    { "npc": "villager", "text": "【剧情台词】" },
    { "npc": "iron_golem", "text": "【剧情台词】" }
  ],

  "theory": "【50-80 字故事摘要】本关发生了什么。不写概念定义。",

  "key_points": [
    "【知识点 1 — 只写关键词组，如「class 声明自定义类型」】",
    "【知识点 2】"
  ],

  "initial_code": [],

  "blanks": [],

  "goal": {
    "checks": []
  },

  "hints": [
    "【方向性提示 — 引导思考，不透露答案】",
    "【工具性提示 — 提示用到的语法】",
    "【模板性提示 — 给出接近答案的模板】"
  ],

  "success_message": "【通关文案 — 必须提到 Canvas 上发生了什么变化 + 剧情收尾】",

  "failure_guidance": "【失败文案 — 叙事化的错误原因】",

  "code_line_notes": [
    "【与 initial_code 行数一致，说明每行在故事中的意义】"
  ],

  "estimated_time_min": 5
}
```

### 3.2 `initial_code` 代码模板规范

- 必须包含 `int main() {` 和 `}` 包裹作用域
- `____` 表示挖空位置，每行出现的顺序与 `blanks` 数组顺序一致
- 没有挖空的代码行直接给出
- 缩进用 4 空格
- 代码行数 ≤ 12 行

### 3.3 `blanks` 挖空定义

```json
{
  "line": 0,
  "type": "text_input",
  "placeholder": "【提示占位词】",
  "width_chars": 10,
  "validator": "cpp_keyword"
}
```

可选 `validator` 值：`cpp_keyword` | `cpp_access` | `identifier` | `number` | `any`

### 3.4 `goal.checks` 判定条件

可用类型：`class_exists` | `class_has_member` | `instance_exists` | `member_value_equals` | `instance_count` | `no_errors`

每个 check 都是一个独立判定，**全部通过**才通关。最后一个 check 必须是 `{"type": "no_errors"}`。

## 四、Canvas 可视化指引

Canvas 的渲染能力（供你设计情节时参考）：

| 引擎事件 | Canvas 表现 | 可用于表现什么剧情 |
|----------|------------|-----------------|
| 类定义 | Class 区出现绿色卡片（类名 + 成员列表 + 🔓/🔒 图标） | 小C创建了家家的「类别档案」 |
| 对象实例化 | Instance 区出现蓝色卡片（变量名 + 值表格） | 家家在世界上出现 |
| 成员赋值 | 实例卡片上的值闪烁更新 | 家家的能量值变化、颜色变化 |
| private 越权 | 成员标红 + ⚠ | 小C的手被弹开 |
| 作用域退出 | 所有卡片渐隐消失 | 剧情转场 |

## 六、示例：第一关的剧情骨架（供参考，可自由改写）

**剧情**：小C在矿洞挖矿时，镐子敲到了一块发着暗紫色微光的黑色方块。方块很轻，触感温热。小C犹豫了很久，把它带回了家。铁傀儡站在门口，看着小C。

```
villager: （小C捧着一块发光的黑色方块，手在微微发抖）
铁傀儡: 「这是什么？没见过的东西。」
villager: 「我...我也不知道。但是它好暖和。我想叫它『家家』。」
铁傀儡: 「给它起名字之前，先给它建立一份档案。告诉世界它是什么。」
```

**玩家的操作**：帮小C写一份「类别档案」— 定义一个 class，命名为家家，创建一个家家对象。

**代码模板**：
```cpp
int main() {
    ____ JIAJIA {        // 小C翻开一本书，打算写一张类别档案
    public:
        int color;        // 家家的颜色
    };
    JIAJIA jiajia;       // 小C给家家起了名字，家家诞生了
}
```

**Canvas 变化**：Class 区出现绿色 `class JIAJIA` 卡片 → Instance 区出现蓝色 `jiajia` 实例卡片。铁傀儡点头：「档案建好了。现在家家是一个被世界承认的存在了。」
